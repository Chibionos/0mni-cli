# Refactor Skill

When asked to refactor code:

1. Understand the current code structure and its tests
2. Identify the refactoring pattern (extract method, rename, move, simplify)
3. Make changes incrementally - one refactoring at a time
4. After each change, verify tests still pass if available
5. Preserve public API contracts unless explicitly asked to change them
6. Explain each refactoring decision
